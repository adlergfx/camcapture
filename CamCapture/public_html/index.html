Available Routes

GET {filename}

If the server is build with serving static files, it will deliver all files in its working directory. The default working directory is public_html. You can pass a different working directory as first and only command line argument to the application. Be carefull, to set the working directory wrong.
GET /capture/{prefix?} Will capture a camera image and returns the filename of the capture. On GET an optinal prefix can be set by route parameter.
POST /capture Will capture a camera image and returns the filename of the capture. The prefix can be set in a json Object of the body in the format {"prefix": "example"}
GET /list/{prefix?}/{from?}/{to?}

Will return a list of available captures. If prefix is set, only matching files wil be returned If from is set only captures equal or later that date are returned. If from and to is set only captures in that interval are returned.

Nummerical parameters are interpreted as dates. The first will be from, the second to. The first non-Numerical Argument is assumed as prefix.

Therefor /list/20260101/abs will be interpreted as from beeing 20260101 with prefix abs. The route /list/123/456/789 will be interpreted with from=123 to=456 and no prefix set!

If you want a numerical prefix, include a non numerical character in the prefix.
GET /image/{capturefile}

Will deliver the captured file as image e.g to view it in the browser or Error 404 if file does not exists.

Example: /image/capture20260101120000.png
